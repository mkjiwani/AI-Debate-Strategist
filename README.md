# AI-Debate-Strategist
A multi-layer AI reasoning application that analyzes cricket debate topics through 5 progressive reasoning layers, with full telemetry tracking and optimization insights.



## Features

- **5-Layer Multi-Layer Reasoning** — Context Analysis → Argument Builder → Counter Argument → Self-Critique → Final Strategy. Each layer produces new reasoning, not repetition.
- **AI Telemetry Dashboard** — Tracks input tokens, output tokens, total tokens, and latency for every API call.
- **Optimization Insights** — Automated analysis of token usage efficiency and latency patterns.
- **Interactive Visualizations** — Bar charts for token usage and latency breakdown per layer.
- **Model Selection** — Switch between gpt-4o-mini, gpt-4o, gpt-4-turbo, and gpt-3.5-turbo from the UI.

## Architecture

```
app.py            → Streamlit UI (entry point)
reasoning.py      → Multi-layer reasoning engine (orchestrates 5 layers)
prompts.py        → Prompt templates for each reasoning layer
telemetry.py      → Telemetry tracking, aggregation, and optimization analysis
.env.example      → Environment variable template
requirements.txt  → Python dependencies
```

## Prerequisites

- Python 3.9+
- An OpenAI API key ([get one here](https://platform.openai.com/api-keys))

## Setup

1. **Clone or download** this folder.

2. **Install dependencies:**
   ```bash
   pip install -r requirements.txt
   ```

3. **Configure environment variables:**
   ```bash
   cp .env.example .env
   ```
   Open `.env` and set your `OPENAI_API_KEY`.

4. **Run the app:**
   ```bash
   streamlit run app.py
   ```

5. The app opens in your browser. Select a topic, choose a model, and click **Run Debate Analysis**.

## Environment Variables

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `OPENAI_API_KEY` | Yes | — | Your OpenAI API key |
| `DEFAULT_MODEL` | No | `gpt-4o-mini` | Default model shown in the UI |
| `TEMPERATURE` | No | `0.7` | Temperature for AI responses (0.0–1.0) |

> **Note:** You can also enter your API key directly in the sidebar if you prefer not to use a `.env` file.

## How It Works

The app chains 5 reasoning layers sequentially. Each layer receives the output of the previous layer(s) as context:

| Layer | Purpose | Guiding Question |
|-------|---------|-----------------|
| 1. Context Analysis | Understand the problem, identify key factors | *What is the problem and what influences it?* |
| 2. Argument Builder | Take a clear position with structured logic | *What is my position and why?* |
| 3. Counter Argument | Generate the strongest opposing viewpoint | *What is the strongest opposing view?* |
| 4. Self-Critique | Identify weaknesses, gaps, or bias | *Where is my argument weak?* |
| 5. Final Strategy | Synthesize into a balanced conclusion | *What is the most informed answer?* |

## Debate Topics

1. **Strike rate vs average** — Is strike rate more important than batting average in modern cricket?
2. **T20 leagues impact** — Have T20 leagues had a net positive or negative impact on international cricket?

## Telemetry

The app tracks and displays:
- **Input tokens** — tokens sent to the model per layer
- **Output tokens** — tokens generated by the model per layer
- **Total tokens** — sum of input + output
- **Latency** — wall-clock time per API call (milliseconds)
- **Throughput** — output tokens per second
- **Optimization insights** — automated suggestions for prompt efficiency
